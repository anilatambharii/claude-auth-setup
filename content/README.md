## Suggested launch order

A staggered launch outperforms a single-day blast. Suggested sequence over ~7 days:

| Day | Channel | Why this slot |
|---|---|---|
| **Day 1 (Tue/Wed)** | Medium + dev.to + GitHub README polish | Anchor content. All other posts will link here. |
| **Day 1, +2hr** | Show HN | Submit between 7–10am ET on a weekday. Engage fast. |
| **Day 2** | LinkedIn + X thread | Drive professional + dev social. Link to Medium. |
| **Day 3** | Reddit (one sub) | r/ClaudeAI is highest-fit. Don't crosspost same day. |
| **Day 5** | Reddit (second sub) | r/programming or r/devops. Different framing. |
| **Day 7** | Substack + portfolio update | Newsletter rounds it out. Portfolio reflects shipped work. |

## Before you publish — checklist

- [ ] Replace `github.com/your-repo/claude-auth-setup` with your actual GitHub URL **everywhere** — every file references it
- [ ] Replace `[Medium link]` and `[Medium]` placeholders with the real Medium URL after publishing it first
- [ ] Confirm the GitHub repo is public, has a clear README, has the LICENSE committed, and the install commands work on a fresh machine
- [ ] Set up GitHub repo "About" + topics (see file 08, section E)
- [ ] Make sure your social handles are consistent across platforms (LinkedIn, X, dev.to, GitHub)
- [ ] If you want analytics, set up a UTM scheme: `?utm_source=linkedin`, `?utm_source=hn`, etc., on your Medium link
- [ ] Have the repo ready to handle 100+ visitors in the first hour after Show HN — landing page should answer "what does this do?" in <5 seconds

## Tone notes per platform

- **Medium / dev.to / Substack** — full voice. Tradeoffs, lessons, mistakes. Long-form readers want depth.
- **LinkedIn** — professional, lead with the business problem (surprise bills), use bullet points, hashtags at end.
- **X** — punchy, technical, no corporate energy. Threads work; single tweets sometimes do too.
- **Reddit** — informal, problem-first, disclose authorship near the end. Don't sound like marketing.
- **Show HN** — terse, honest about tradeoffs and limitations. HN punishes hype and rewards engineering candor.
- **Portfolio** — outcome-focused. What you built, what tradeoffs you made, what shipped.

## What to do with comments

- **HN/Reddit:** engage in the first 2 hours. Top comment shapes ranking.
- **LinkedIn/X:** reply to every comment in the first 24 hours.
- **Medium/dev.to/Substack:** less time-sensitive — within a week is fine.
- **GitHub Issues/PRs:** the highest-leverage feedback. Respond fast, even if just to acknowledge.

Good luck with the launch.
